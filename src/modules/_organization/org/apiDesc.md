# Organization 模块 API 接口文档

## 概述

Organization模块负责组织信息的管理，包括组织的增删改查功能。

## 接口列表

### 1. 获取组织列表
- **路径**: `POST /api/org/list`
- **描述**: 获取组织列表，支持分页和筛选功能
- **认证要求**: 需要有效的访问令牌及相应权限
- **请求参数**:
  - `filter` (可选): 筛选条件对象
    - `regExp` (可选): 正则表达式搜索 (0-50位字符串)
    - `isActive` (可选): 是否激活
    - `isMain` (可选): 是否为主机构
    - `name` (可选): 名称 (1-100位字符串)
    - `unionCode` (可选): 统一代码 (1-30位字符串)
    - `Nation` (可选): 国家 ObjectId
    - `Province` (可选): 省份 ObjectId
    - `City` (可选): 城市 ObjectId
    - `Area` (可选): 地区 ObjectId
  - `options` (可选): 分页和排序选项对象
    - `limit` (可选): 每页数量，默认100，最大值受MAX_HANDLE_ITEM限制
    - `skip` (可选): 跳过的记录数
    - `sort` (可选): 排序对象，格式 { field: 1/-1 }
    - `populate` (可选): 关联填充数组
      - `path` (必填): 填充路径
      - `select` (可选): 选择字段
      - `match` (可选): 过滤条件对象
      - `options` (可选): 附加选项
        - `sort` (可选): 排序
        - `limit` (可选): 限制数量
        - `skip` (可选): 跳过数量
- **响应**:
  - `total`: 总数
  - `items`: 组织列表

### 2. 获取组织详情
- **路径**: `POST /api/org/detail/:id`
- **描述**: 根据ID获取单个组织的详细信息
- **认证要求**: 需要有效的访问令牌及相应权限
- **路径参数**: `id` - 组织ID (必填, ObjectId)
- **请求参数**:
  - `options` (可选): 查询选项对象
    - `populate` (可选): 关联填充数组
      - `path` (必填): 填充路径
      - `select` (可选): 选择字段
      - `match` (可选): 过滤条件对象
      - `options` (可选): 附加选项
- **响应**: `item` - 组织详细信息

### 3. 创建组织
- **路径**: `POST /api/org/add`
- **描述**: 创建新组织
- **认证要求**: 需要有效的访问令牌及添加权限
- **请求参数** (可选):
  - `isActive` (可选): 是否激活
  - `sort` (可选): 排序 (数字，最小值为0)
  - `unionCode` (必填): 统一代码 (2-30位字符串)
  - `name` (必填): 组织名称 (2-100位字符串)
  - `nickname` (必填): 昵称 (1-50位字符串)
  - `phone` (可选): 电话 (7-20位字符串)
  - `email` (可选): 邮箱 (5-100位字符串)
  - `website` (可选): 网站 (5-200位字符串)
  - `address` (可选): 地址 (5-200位字符串)
  - `isMain` (可选): 是否为主机构
  - `Nation` (可选): 国家 ObjectId
  - `Province` (可选): 省份 ObjectId
  - `City` (可选): 城市 ObjectId
  - `Area` (可选): 地区 ObjectId
- **响应**: `item` - 创建的组织信息

### 4. 更新组织信息
- **路径**: `POST /api/org/edit/:id`
- **描述**: 根据ID更新组织信息
- **认证要求**: 需要有效的访问令牌及编辑权限
- **路径参数**: `id` - 组织ID (必填, ObjectId)
- **请求参数** (可选):
  - `isActive` (可选): 是否激活
  - `sort` (可选): 排序 (数字，最小值为0)
  - `unionCode` (可选): 统一代码 (2-30位字符串)
  - `name` (可选): 组织名称 (2-100位字符串)
  - `nickname` (可选): 昵称 (1-50位字符串)
  - `phone` (可选): 电话 (7-20位字符串)
  - `email` (可选): 邮箱 (5-100位字符串)
  - `website` (可选): 网站 (5-200位字符串)
  - `address` (可选): 地址 (5-200位字符串)
  - `isMain` (可选): 是否为主机构
  - `Nation` (可选): 国家 ObjectId
  - `Province` (可选): 省份 ObjectId
  - `City` (可选): 城市 ObjectId
  - `Area` (可选): 地区 ObjectId
- **响应**: `item` - 更新后的组织信息

### 5. 获取当前用户所属组织信息
- **路径**: `POST /api/org/self`
- **描述**: 获取当前登录用户所属组织的信息
- **认证要求**: 需要有效的访问令牌和用户授权
- **请求参数**:
  - `options` (可选): 查询选项对象
    - `populate` (可选): 关联填充数组
      - `path` (必填): 填充路径
      - `select` (可选): 选择字段
      - `match` (可选): 过滤条件对象
      - `options` (可选): 附加选项
- **响应**: 当前用户所属组织信息

## 权限说明

- `readPermission`: 需要读取权限才能访问列表和详情接口
- `addPermission`: 需要添加权限才能创建组织
- `editPermission`: 需要编辑权限才能更新组织信息

## 响应格式

所有接口均采用统一的响应格式：

```json
{
  "code": 200,
  "success": true,
  "message": "操作成功",
  "data": {}
}
```
