# 教室管理模块 (Room Module)

## 概述
教室管理模块负责处理培训机构中教室资源的管理，包括教室信息的增删改查等功能。

## 功能说明

### 1. 获取教室列表 (`/list`)
- **方法**: POST
- **权限**: 管理员或具有相应权限的用户
- **功能**: 获取教室列表信息，支持按名称、容量、位置、状态等条件筛选
- **参数**:
  - filter: 筛选条件对象
    - name: 教室名称
    - capacity: 容量
    - location: 位置
    - status: 状态
    - isActive: 激活状态
  - options: 查询选项对象
    - limit: 限制数量
    - skip: 跳过数量
    - sort: 排序方式

### 2. 获取教室详情 (`/detail/:id`)
- **方法**: POST
- **权限**: 管理员或具有相应权限的用户
- **功能**: 获取指定教室的详细信息
- **参数**:
  - id: 教室ID (路径参数)

### 3. 创建教室 (`/add`)
- **方法**: POST
- **权限**: 管理员
- **功能**: 创建新的教室信息
- **参数**:
  - name: 教室名称 (必填)
  - capacity: 容量 (选填)
  - location: 位置 (选填)
  - description: 描述 (选填)
  - status: 状态 (选填)
  - isActive: 激活状态 (选填)

### 4. 更新教室信息 (`/edit/:id`)
- **方法**: POST
- **权限**: 管理员
- **功能**: 更新指定教室的信息
- **参数**:
  - id: 教室ID (路径参数)
  - name: 教室名称 (选填)
  - capacity: 容量 (选填)
  - location: 位置 (选填)
  - description: 描述 (选填)
  - status: 状态 (选填)
  - isActive: 激活状态 (选填)

## 权限控制

### 学生权限
- 学生无法访问教室管理功能

### 普通用户权限
- 普通用户需要是管理员(manager)角色才能访问教室功能
- 普通管理员只能操作自己机构下的教室

### 管理员权限
- 系统管理员(admin)可以跨机构操作教室
- 普通管理员(manager)只能操作自己机构下的教室

## 数据模型

### Room (教室)
- `_id`: 教室唯一标识符
- `name`: 教室名称 (必填)
- `capacity`: 教室容量 (默认值: 10)
- `location`: 位置描述 (如"一楼东侧")
- `description`: 教室备注 (设备情况等)
- `status`: 状态 ('available'-可用, 'in_use'-使用中, 'maintenance'-维护中) 默认值: 'available'
- `isActive`: 是否激活 (默认值: true)
- `Org`: 所属机构ID (必填)
- `createdBy`: 创建者ID
- `updatedBy`: 更新者ID
- `timestamps`: 创建时间和更新时间

## 错误码说明
- `401`: 未认证
- `403`: 无权限
- `404`: 教室不存在
- `500`: 服务器内部错误
