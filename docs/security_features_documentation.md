# 系统安全增强功能说明

## 1. 并发登录预防机制

### 1.1 功能概述
系统现在支持防止同一账号在多个设备上同时登录的功能。当用户在新设备登录时，之前设备上的会话将自动失效。

### 1.2 实现原理
- **新增字段**: 在 `Account` 模型中添加了 `currentSessionId` 字段
- **会话管理**: 登录时生成唯一会话ID，存储在数据库中
- **验证机制**: 每次请求时验证当前会话ID是否与数据库中的一致
- **安全登出**: 登出时清空 `currentSessionId`，使该会话失效

### 1.3 技术实现
- **Account 模型** ([Account.model.js](server/src/models/authorization/Account.model.js)):
  - 新增 `currentSessionId` 字段
  - 刷新令牌采用 Argon2 哈希存储，提升安全性

- **登录服务** ([login.service.js](server/src/modules/_authorization/auth/login.service.js)):
  - 登录时生成唯一会话ID
  - 刷新令牌时更新会话ID
  - 提供 `forceLogoutAllDevices` 方法强制登出所有设备

- **JWT 工具** ([JwtUtil.js](server/src/utils/JwtUtil.js)):
  - 验证令牌时检查会话有效性
  - 统一的载荷结构

- **认证中间件** ([auth.js](server/src/middlewares/auth.js)):
  - 请求时验证会话状态
  - 会话失效时返回明确错误信息

- **会话验证工具** ([sessionValidator.js](server/src/utils/sessionValidator.js)):
  - 集中的会话验证逻辑
  - 提供多种会话管理方法

## 5. 使用示例

### 5.1 正常登录流程
1. 用户提交账号密码
2. 系统验证身份
3. 生成新的会话ID
4. 存储会话信息
5. 返回访问令牌

### 5.2 并发登录场景
1. 用户A在设备1登录 → 生成会话ID1
2. 同一用户在设备2登录 → 生成会话ID2，旧会话ID1失效
3. 设备1访问系统 → 会话ID不匹配 → 自动登出

### 5.3 会话管理
```javascript
// 强制登出所有设备
await LoginService.forceLogoutAllDevices(userId);
```

## 6. 错误处理

系统提供了详细的错误信息：
- `会话已失效或已在其他设备登录，请重新登录`
- `无效的刷新令牌 - 会话已过期或被其他设备登录`
- 标准化的错误日志记录

## 7. 注意事项

- 会话ID是随机生成的字符串，保证唯一性
- 登出操作会清空当前会话，但不影响账号本身
- 管理员可强制登出指定用户的全部会话
- 会话验证对性能影响很小，使用了高效的查询方式
